# GitHub
GitHub é uma plataforma de hospedagem de código-fonte e controle de versão que utiliza o sistema Git. Ele permite que os desenvolvedores colaborem em projetos, compartilhem código e gerenciem suas versões de forma eficiente. O GitHub oferece uma variedade de recursos, como repositórios, issues, pull requests e Gists, que facilitam a colaboração e o gerenciamento de projetos de software.

## Gist
Um Gist no GitHub é uma ferramenta que permite compartilhar trechos de código, notas ou arquivos de forma simples e eficiente. Ele funciona como um repositório Git, o que significa que você pode clonar, bifurcar e acompanhar o histórico de alterações. É ideal para armazenar pequenos pedaços de código ou informações que não justificam a criação de um repositório completo.
### Criando um Gist
1. Acesse [gist.github.com](https://gist.github.com/) ou "https://gist.github.com/seuusername" .
2. Faça login com sua conta do GitHub.
3. Clique em "New Gist".
4. Preencha o título e a descrição do Gist.
5. Adicione o conteúdo do Gist, seja código ou texto.
6. Escolha se o Gist será público ou privado.
7. Clique em "Create public gist" ou "Create secret gist" para finalizar.

### Gerenciando Gists- Você pode editar, excluir ou compartilhar seus Gists a qualquer momento.
- Para editar um Gist, acesse a página do Gist e clique em "Edit".
- Para excluir um Gist, acesse a página do Gist e clique em "Delete".
- Para compartilhar um Gist, copie a URL do Gist e compartilhe com outras pessoas

## Repositórios
Um repositório no GitHub é um espaço onde você pode armazenar, organizar e gerenciar projetos de código-fonte. Ele permite que você controle as versões do seu código, colabore com outros desenvolvedores e compartilhe seu trabalho com a comunidade. Os repositórios podem ser públicos ou privados, dependendo das suas necessidades de compartilhamento e colaboração.
### Criando um Repositório
1. Acesse [github.com](https://github.com/) e faça login com sua conta.
2. Clique em "New repository".
3. Preencha o nome do repositório, a descrição e escolha se ele será público ou privado.
4. Clique em "Create repository" para finalizar.

### Gerenciando Repositórios
- Para editar um repositório, acesse a página do repositório e clique em "Settings".
- Para excluir um repositório, acesse a página do repositório, clique em "Settings" e role para baixo até a seção "Danger Zone", onde você encontrará a opção "Delete this repository".
- Para compartilhar um repositório, copie a URL do repositório e compartilhe com outras pessoas.
- Para colaborar em um repositório, você pode adicionar colaboradores ou criar um fork do repositório para trabalhar em suas próprias alterações antes de enviar um pull request para o repositório original.

### Colaboração em Repositórios
- Para colaborar em um repositório, você pode adicionar colaboradores ou criar um fork do repositório para trabalhar em suas próprias alterações antes de enviar um pull request para o repositório original.
- Para adicionar colaboradores, acesse a página do repositório, clique em "Settings", depois em "Manage access" e adicione os colaboradores desejados.
- Para criar um fork, acesse a página do repositório e clique em "Fork". Isso criará uma cópia do repositório em sua conta, onde você pode fazer alterações e, posteriormente, enviar um pull request para o repositório original.
- Para enviar um pull request, acesse a página do repositório original, clique em "Pull requests" e depois em "New pull request". Selecione as alterações que deseja enviar e clique em "Create pull request" para finalizar.

## Issues
As Issues no GitHub são uma ferramenta de gerenciamento de tarefas e acompanhamento de bugs. Elas permitem que os desenvolvedores e colaboradores registrem problemas, solicitem melhorias ou discutam ideias relacionadas a um projeto. As Issues podem ser atribuídas a membros da equipe, etiquetadas para organização e fechadas quando resolvidas.

### Criando uma Issue
1. Acesse a página do repositório onde deseja criar a Issue.
2. Clique na aba "Issues".
3. Clique em "New issue".
4. Preencha o título e a descrição da Issue.
5. Clique em "Submit new issue" para finalizar.

### Gerenciando Issues
- Para editar uma Issue, acesse a página da Issue e clique em "Edit".
- Para fechar uma Issue, acesse a página da Issue e clique em "Close issue".
- Para reabrir uma Issue, acesse a página da Issue e clique em "Reopen issue".
- Para atribuir uma Issue a um membro da equipe, acesse a página da Issue, clique em "Assignees" e selecione o membro desejado.
- Para etiquetar uma Issue, acesse a página da Issue, clique em "Labels" e selecione as etiquetas desejadas para organizar a Issue.
- Para comentar em uma Issue, acesse a página da Issue e utilize a seção de comentários para discutir o problema ou fornecer informações adicionais.
- Para fechar uma Issue automaticamente, você pode incluir palavras-chave como "fixes", "closes" ou "resolves" seguidas do número da Issue no commit message ou na descrição do pull request. Por exemplo: "fixes #123" fechará a Issue número 123 quando o commit for mesclado.
